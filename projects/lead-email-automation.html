The Problem

A sales team was manually copying leads from a spreadsheet, writing individual emails, and tracking responses in notes. It was exhausting, inconsistent, and they could only reach about 20 leads per day. They needed to scale to hundreds without hiring more staff or losing the personal touch.

The Solution

I built a Python automation pipeline that handles the entire outreach workflow from start to finish:

  • Reads leads from CSV — name, email, company, role, notes. No special database required to get started
  • Personalizes each email — uses lead data to customize subject lines, greetings, references to their company, and tailored CTAs
  • Sends in batches — configurable delays between emails to stay under provider rate limits and avoid spam flags
  • Retry logic for failures — if SMTP drops a message, it retries up to 3 times before logging it as failed
  • Open tracking — lightweight tracking pixel records when each lead opens the email
  • Bounce handling — invalid emails are caught and logged so the lead list stays clean
  • Summary report — generates a CSV with sent/failed/opened stats after each run

The Core Automation

def send_campaign(
    leads_file: Path,
    template: str,
    smtp_config: dict,
    batch_size: int = 50
) -> CampaignReport:

    leads = read_leads(leads_file)
    report = CampaignReport(total=len(leads))

    for i, lead in enumerate(leads):
        # Personalize subject + body from lead data
        email = render_template(template, lead)

        # Respect rate limits — send in batches
        if i > 0 and i % batch_size == 0:
            log.info("Batch complete. Pausing 5 min...")
            time.sleep(300)

        # Send with retry on failure
        success = send_with_retry(
            to=lead.email,
            subject=email.subject,
            body=email.body,
            smtp=smtp_config
        )
        report.record(send=success, lead=lead)

    report.export("campaign_report.csv")
    return report

Tech Stack

  • Python 3.11 — Automation engine
  • smtplib / email — Python stdlib email sending
  • Jinja2 — Email template rendering with personalization
  • Pandas — CSV reading and data validation
  • Pillow — Tracking pixel generation
  • Flask (optional) — Lightweight dashboard for viewing campaign stats
  • Schedule / cron — Automated recurring campaigns

The Result

What used to require days of manual copy-paste and email composition now runs as a single Python command. 500+ personalized, tracked emails go out in organized batches with zero manual intervention. The team gets a clear report showing exactly who received, opened, and bounced — making follow-ups targeted instead of guesswork.
